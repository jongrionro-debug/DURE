# 큰글씨 운영 서비스 생성 흐름

이 문서는 현재 PRD를 바탕으로, `큰글씨`라는 기관이 처음 이 제품에 들어와 운영 서비스를 만드는 흐름을 설명한다.

## 한 줄 요약

큰글씨의 첫 기관 운영자가 회원가입 후 로그인하면, `기관 이름`과 `첫 번째 마을 이름`만 입력해서 자기 기관 전용 운영 시스템의 뼈대를 만든다. 그 다음 `첫 프로그램 만들기`부터 시작해 수업, 강사, 참여자 명단, 세션을 차례로 채우고, 이후 강사는 배정된 세션에서 출석과 교육일지를 제출한다.

## 전체 흐름

1. 큰글씨의 최초 기관 운영자가 회원가입을 한다.
2. 로그인한다.
3. 이 계정은 아직 어떤 기관에도 연결되지 않았으므로 `기관 생성 온보딩`으로 들어간다.
4. 화면에는 단순한 입력 중심 레이아웃이 보인다.
5. 운영자는 `기관 이름 = 큰글씨`를 입력한다.
6. 운영자는 `첫 번째 마을 이름`을 입력한다.
7. 시스템은 큰글씨 전용 운영 시스템의 초기 뼈대를 생성한다.
8. 이 최초 생성자는 자동으로 `큰글씨 기관의 최고 운영자`가 된다.
9. 생성 직후 `생성 완료 요약` 화면이 나온다.
10. 이 화면은 큰글씨와 첫 번째 마을이 만들어졌다는 것을 보여준다.
11. 이 화면의 핵심 다음 액션은 `첫 프로그램 만들기`다.

## 생성 직후 상태

이 시점에 이미 만들어진 것:

- 기관: `큰글씨`
- 첫 번째 마을
- 기관 운영자 권한 구조의 시작점

이 시점에 아직 비어 있는 것:

- 프로그램
- 수업
- 강사
- 참여자 명단
- 세션

즉, 생성 직후에는 `빈 운영 시스템`이 허용된다. 대신 운영자는 다음 단계에서 필요한 운영 정보를 하나씩 채워 넣는다.

## 운영자가 다음으로 하는 일

### 1. 첫 프로그램 만들기

운영자는 운영 기본정보 설정으로 이동한다.

여기서 가장 먼저 하게 되는 일은 `첫 프로그램 만들기`다.

예:
- 프로그램 이름: `2026 봄 마을 돌봄 프로그램`

기본 프로그램은 자동으로 생기지 않는다. 운영자가 직접 만들고 이름을 정한다.

### 2. 수업 만들기

프로그램을 만든 뒤, 그 프로그램 아래 실제 운영할 수업을 만든다.

예:
- 독서 수업
- 글쓰기 수업
- 미술 수업

각 수업은 마을과 연결될 수 있다.

### 3. 강사 초대

운영자는 강사를 이메일 초대 링크 방식으로 초대한다.

강사는 초대를 수락한 뒤 큰글씨 기관에 소속된다.

현재 MVP에서는 한 사용자가 여러 기관에 동시에 속하지 않으므로, 이미 다른 기관에 속한 이메일은 초대가 실패하고 안내 메시지가 나온다.

### 4. 강사 배정

운영자는 초대한 강사를 특정 수업 또는 세션에 배정한다.

예:
- 김OO 강사 -> 독서 수업 담당
- 박OO 강사 -> 미술 수업 담당

### 5. 참여자 명단 등록

운영자는 각 수업의 출석 대상이 될 참여자 명단을 등록한다.

예:
- 독서 수업 참여자 12명
- 미술 수업 참여자 9명

### 6. 세션 생성

운영자는 `한 날짜의 한 수업 세션` 기준으로 실제 운영 세션을 미리 생성한다.

예:
- 2026-03-10 / 성북 1마을 / 독서 수업 / 김OO 강사
- 2026-03-10 / 성북 1마을 / 미술 수업 / 박OO 강사

이때 세션 생성 시점의 참여자 명단은 세션에 복사되어 고정된다. 나중에 원본 명단이 바뀌어도 이미 만들어진 과거 세션의 출석 대상은 자동으로 바뀌지 않는다.

## 강사의 실제 사용 흐름

1. 강사가 로그인한다.
2. 강사는 자기에게 배정된 세션만 본다.
3. 오늘 담당 세션을 선택한다.
4. 전자 출석을 입력한다.
5. 교육일지를 작성한다.
6. 필요하면 첨부 문서를 올린다.
7. 최종 제출한다.

제출 후에도 수정은 가능하다.

- `submitted_at`은 최초 제출 시각으로 유지된다.
- 이후 수정이 일어나면 `updated_at`만 바뀐다.
- 최근 제출 기록과 최근 수정 기록은 운영자 대시보드에서 분리해서 볼 수 있다.

## 운영자가 보는 대시보드

큰글씨 기관 운영자는 대시보드에서 아래를 본다.

- 오늘 제출 현황
- 미제출/완료 요약
- 최근 제출 기록
- 최근 수정 기록
- 빠른 이동 액션

여기서:

- `최근 제출 기록`은 처음 제출된 세션 중심
- `최근 수정 기록`은 강사 또는 운영자가 이후 수정한 세션 중심

즉 운영자는 `새로 들어온 기록`과 `방금 변경된 기록`을 분리해서 이해할 수 있다.

## 이 흐름의 핵심 의미

이 제품은 처음부터 모든 정보를 한 번에 입력하게 하지 않는다.

대신:

1. 기관과 첫 마을만으로 운영 시스템의 뼈대를 만든다.
2. 운영자는 그 안에서 프로그램, 수업, 강사, 명단, 세션을 하나씩 채운다.
3. 강사는 배정된 세션에서 기록을 제출한다.
4. 운영자는 제출과 수정 현황을 관리한다.

즉 `생성 -> 설정 -> 운영 -> 관리`의 순서로 흘러가는 구조다.
